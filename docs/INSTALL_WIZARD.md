# Install Wizard

Эта инструкция описывает только первый запуск сервера через web wizard после установки Family Messenger на новый VPS.

Если нужен полный серверный deploy workflow, сначала смотри `docs/DEPLOY_RUNBOOK.md`. Если нужен общий обзор infra-скриптов, смотри `infra/README.md`.

## Когда Нужен Этот Wizard

Wizard нужен один раз на чистой базе данных.

Он не ставит Docker, Java, Caddy или `systemd`. Это делает установочный скрипт:

```bash
curl -fsSL https://raw.githubusercontent.com/hram/family-messenger/main/infra/install.sh | bash
```

После этого сервер уже поднят, а wizard делает только прикладочную инициализацию:

- задаёт master password для админ-панели
- создаёт семью
- создаёт первых участников
- генерирует invite-коды для входа

Важно:

- `install.sh` специально пишет `DB_BOOTSTRAP_SCHEMA=false` и `DB_SEED_ON_START=false`
- схема БД создаётся через `schema.sql`
- семья и invite-коды создаются именно через `POST /api/setup/bootstrap`
- после успешного bootstrap повторно пройти wizard на той же БД нельзя

## Перед Началом

Нужно иметь:

- новый сервер с Ubuntu 24.04
- доступ по `ssh`
- установленный Family Messenger через `infra/install.sh`
- URL, который напечатал установщик, обычно `http://<server-ip>:8080`

Проверь, что backend отвечает:

```bash
curl http://<server-ip>:8080/api/health
```

Ожидается успешный ответ `200 OK`.

Дополнительно можно проверить, что система ещё не инициализирована:

```bash
curl http://<server-ip>:8080/api/setup/status
```

На чистом сервере ожидается `initialized=false`.

## Порядок Прохождения Wizard

### Шаг 1. Открыть Web Клиент

Открой в браузере URL, который напечатал `install.sh`, например:

```text
http://<IP>:8080
```

На web-клиенте для чистой БД автоматически откроется setup wizard.

### Шаг 2. Указать Server URL И Master Password

На первом шаге wizard:

- проверь `Server URL`
- задай `master password`
- повтори его во втором поле подтверждения

Что важно про master password:

- это пароль для панели администрирования, а не invite-код
- через него потом подтверждаются административные действия
- в базе не хранится открытый пароль, сохраняется только хэш
- если пароль потерян, восстановить его штатно нельзя

Практическое правило:

- сохрани пароль в менеджер паролей до перехода на следующий шаг

### Шаг 3. Заполнить Семью И Участников

На втором шаге wizard:

- укажи название семьи
- добавь участников кнопкой `Добавить участника`
- для каждого участника задай имя
- выбери роль `Parent` или `Child`
- для родителей при необходимости включи флаг администратора

Рекомендуется:

- сделать хотя бы одного взрослого участника администратором
- использовать понятные отображаемые имена, которые потом увидят все члены семьи

После заполнения нажми `Create`.

### Шаг 4. Сохранить Invite-Коды

После успешного bootstrap wizard покажет список участников и invite-коды.

Это ключевой результат первого запуска. Для каждого участника будет создан отдельный invite-код.

На этом экране нужно:

- сохранить invite-коды
- раздать каждому участнику его собственный код
- при необходимости показать QR-код

Важно:

- если закрыть страницу и забыть коды, сам wizard повторно не откроется
- дальше управлять участниками нужно уже через обычную админ-панель приложения

### Шаг 5. Войти В Приложение

После сохранения invite-кодов нажми `Войти в приложение`.

Дальше каждый участник может авторизоваться своим invite-кодом.

## Что Проверить После Wizard

Минимальная проверка с сервера:

```bash
curl http://<server-ip>:8080/api/setup/status
curl http://<server-ip>:8080/api/health
```

После успешного bootstrap у `setup/status` ожидается `initialized=true`.

Если нужен более прикладочный smoke-check, проверь:

- web-клиент открывается по публичному URL
- invite-кодом можно выполнить первый вход
- у администратора открываются админ-функции после ввода master password

## Если Wizard Не Открывается

Проверь по порядку:

1. Установщик действительно завершился без ошибки.
2. `systemctl is-active family-messenger-backend` возвращает `active`.
3. `docker ps` показывает postgres-контейнер для нужного контура.
4. `curl http://<server-ip>:8080/api/health` отвечает успешно.
5. `curl http://<server-ip>:8080/api/setup/status` не возвращает `initialized=true`.

Если `initialized=true`, значит эта БД уже была инициализирована и wizard больше не должен показываться.

## Как Начать Заново На Чистом Сервере

Если первый запуск был испорчен и нужно полностью пройти wizard заново, недостаточно просто перезагрузить страницу. Нужно удалить текущую установку или как минимум сбросить БД текущего контура.

Самый прямой путь для полного сброса `prod`:

```bash
curl -fsSL https://raw.githubusercontent.com/hram/family-messenger/main/infra/uninstall.sh | bash
curl -fsSL https://raw.githubusercontent.com/hram/family-messenger/main/infra/install.sh | bash
```

Для `dev` контура использовать только `infra/uninstall-dev.sh` и `infra/install-dev.sh`.

Важно:

- не путай `prod` и `dev`
- у них разные install root, config root, `systemd` unit, порты и postgres volume
- перед любым ручным действием на семейном сервере сверяйся с `docs/DEPLOY_RUNBOOK.md`
